SQL - Grouping data (GROUP BY)

Grouped Aggregation & GROUP BY clause

  • Grouped Aggregation allows us to collate rows in a result set based on some criteria and apply some operations or further filtering on collated/grouped result sets.
  • A grouping field is selected to group the rows. The rows are sorted by the grouping field.
  • Rows with the same value for the grouping field are treated as a "group".
  • The GROUP BY clause is added to the SELECT statement in order to group the data from the SELECT table(s) and produces a single summary row for each group.
  • Usually a statistical function is also used and applied to each group (e.g: SUM, AVG, COUNT, etc.)

GROUP BY - HAVING clause

  • The HAVING clause is used alongside the GROUP BY clause, with a purpose to filter or restrict groups created by the GROUP BY clause subject to some condition.
  • It operates like a WHERE clause, but is applied to the grouping value.
  • While WHERE clause is applied to each row before the grouping operation is done, HAVING clause is applied after the grouping query and operates on the calculated grouping value (before it is displayed).
