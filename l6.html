SQL - Nested queries

Nested Query Introduction

  • In this section, we examine the use of a complete SELECT statement embedded within another SELECT statement.
  • The results of this nested (inner) SELECT statement are used in the outer statement to help determine the contents of the final result.

Three types of nested query

  • A scalar subquery:
    • It returns a single column and a single row; that is, a single value.
    • In principle, a scalar subquery can be used whenever a single value is needed.
    • Example: The inner query generates a single value, which is, in turn, used in a WHERE clause as part of a condition.
  • A single-column subquery:
    • It returns multiple rows, but again only a single column.
    • A single-column subquery can be used where a set of (similar-typed) values are expected.
    • Example: The inner query produces a set of values, which are used with IN clause.
  • A table subquery:
    • It returns one or more columns and multiple rows.
    • A table subquery can be used whenever a table is needed.
    • Example: The inner query produces a temporary table within a FROM clause.

Execution order of nested queries

  1. Inner query is processed first, producing values (can be a single value, mutiple values or a table).
  2. The outer query uses the resultant values of the inner sub-query for its execution.

Placement of nested queries

  • The nested query can be placed anywhere in the main query
  • The nested query must return an appropriate value(s) to work in a given situation. For example:
    • In the FROM clause (called sub-select): the nested query must return a table.
    • In the WHERE clause: the nested query must return a value or a list of values.
