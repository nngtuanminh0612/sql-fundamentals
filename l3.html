SQL - Retrieving data (SELECT)

Simple SELECT Query

  • Most of the time , users want to extract or search for some data in a table. This is where the SELECT is applied.
  • The syntax for basic SELECT statements is just as simple as:

    SELECT field_name1, field_name2, ...

    FROM table_name

Examples of basic SELECT statements

Changing resultant headers

  • The header of the result, if not specified, will be the same as that in the table selected
  • To change the resultant header, users can give an alias after the field name using the AS keyword

    SELECT field_name1 AS new_header1, field_name2 AS new_header2, ...

    FROM table_name

*NOTE: Some database engines require users to put the field alias in quotes, while some do not, so pay attention to different database-specific dialects for correct syntaxes.

The DISTINCT keyword

  • The DINSTINCT keyword makes SELECT statements omit duplicate rows from the results set.
  • You can also use DISTINCT with two or more columns. In this case the rows of the results set represent all the distinct combinations of the columns in the select list.

Adding conditions for data selection

  • When users want to limit the resultant data, they can put one or more conditions in their queries using a WHERE clause.
  • The SELECT statement with condition(s) will become:

    SELECT field_name1, field_name2, ...

    FROM table_name

    WHERE condition1 AND condition2 AND ...

  • A single SELECT statement can only have one WHERE clause, different conditions need to be combined using the AND keyword

Basic built-in functions in SELECT queries

Typical built-in statistical fuctions for SELECT statements include, but are not limited to:

  • COUNT - for displaying the number of rows that meet specified condition(s) *NOTE: They keywords COUNT and DISTINCT can be used together to count the number of unique results.
  • MAX - for finding the maximum value in a specific column of the records that meet specified condition(s)
  • MIN - for finding the minimum value in a specific column of the records that meet specified condition(s)
  • SUM - for calculating the total of values in a specific column of the records that meet specified condition(s)
  • AVG - for calculating the average of values in a specific column of the records that meet specified condition(s)

Conditions/Selection criteria in the WHERE clause

  • All usual comparison operators can be used (ie: =; >; <; <=; >=; <>)
  • Usual compound operators can be used (ie: AND; OR; NOT)

    • *NOTE: If you use multiple logical operators together in an expression, you need to be mindful of order of operations. In SQL, the precedence of logical operators is NOT, AND, and OR in that order.

  • Brackets can also be used for specifying order of operators
    (eg: NOT(((cond1) AND (cond2)) OR (cond3))

Conditions with Wildcards in the WHERE clause

    Wildcards can be used to specify a pattern of a string in a column.

Conditions using shortcuts in the WHERE clause

    With selection criteria invloving multiple values, users can use some shortcuts to specify conditions faster. These shortcuts include, but are not limited to, IN, BETWEEN ... AND ... , or NOT IN

*A small funny joke about learning SELECT queries

    In reality, 80% of the queries performed by users are for retrieving data. Therefore, by this stage, we have gone through the most basic but essential parts of SQL. Pretty quick, right? SQL seems not too hard!!

    However, unfortunately it will take you 80% of the time from now to learn the remaining 20% of the language!
