Database & RDBMS Introduction

Database vs Database Management System (DBMS) definition?

  • Database: A collection of data stored in a standardised format, designed to be shared by multiple users.
  • Database Management System: Software that defines a database, stores the data, supports a query language, produces reports, and creates data entry scrren.

Why are databases important?

  • Database is an integral part of all information systems
  • Database is an integral part of most websites
  • Almost all software developments involve working with a database
  • Almost all databases use SQL - Structured Query Lanugage

Basic DBMS components?

  • Tables / Entities
  • Records / Tuples / Rows
  • Fields / Attributes / Columns
  • Data Dictionary / Repository
  • DQL - Data Query Laguage
  • DDL - Data Definition Laguage
  • DML - Data Manipulation Laguage
  • DCL - Data Control Laguage

Database structures

  • Relational database

    A relational database is a type of database that stores and provides access to data points that are related to one another. Relational databases are based on the relational model, an intuitive, straightforward way of representing data in tables.

    * RDBMS stands for Relational Database Management System - a DBMS designed specifically for relational databases

    Relational databases are the focus when learning and working with SQL.

  • Hierarchical database

    A hierarchical database model is a data model in which the data are organized into a tree-like structure. The data are stored as records which are connected to one another through links. A record is a collection of fields, with each field containing only one value.

  • Object database

    An object database is a database management system in which information is represented in the form of objects as used in object-oriented programming. The efficiency of this database structure is greatly improved in areas which demand massive amounts of data about one item.

Advantages of database approach for data management?

  • Minimal data redundancy
  • Data consistency
  • Integration of data
  • Efficient sharing of data
  • Enforcement of data
  • Ease of application development
  • Uniform security, privacy and integrity
  • Data independece

Risks and Costs of database approach?

  • Organisational conflicts
  • Development project failure
  • System failure
  • Overhead costs
  • Need for sophisticated personnel
